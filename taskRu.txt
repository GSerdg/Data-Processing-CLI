Задание: Интерфейс командной строки для обработки данных
Описание
Ваша задача — создать инструментарий для обработки данных — интерактивное приложение командной строки, выполняющее различные полезные операции обработки данных. Инструмент должен работать как постоянно запущенный процесс Node.js, принимающий команды.

В отличие от задания по основам Node.js, где вы изучали API по отдельности, здесь вы объедините их в реальный, целостный инструмент с интерактивной навигацией по файловой системе и возможностями обработки данных.

Технические требования
Использование любых сторонних инструментов и библиотек запрещено.
Используйте версию Node.js 24.xx (24.10.0 или выше).
Для повышения эффективности все операции с файлами должны использовать API потоков (не следует считывать целые файлы в память).
По возможности отдавайте предпочтение асинхронному API.
Программа должна представлять собой интерактивную REPL-среду (цикл чтения-выполнения-печати).
Пути к файлам в командах могут быть относительными или абсолютными.
Интерфейс командной строки
Программа запускается через npm-script start:

npm run start
Который запускается:

node src/main.js
Программа должна:

Отображать приветственное сообщение при запуске:Welcome to Data Processing CLI!
Сначала выведите на экран текущий рабочий каталог:You are currently in /path/to/home
Постоянно предлагайте пользователю вводить команды:>
Принимайте команды в формате:<command> [arguments]
Отображение сообщений об ошибках для неизвестных или недопустимых команд без сбоев.
Разрешите пользователям выходить с помощью .exitкоманды илиCtrl+C
При выходе отобразить прощальное сообщение:Thank you for using Data Processing CLI!
После каждой успешной операции снова выведите текущий рабочий каталог.
При запуске программы рабочим каталогом должен быть домашний каталог пользователя.
Если команда неизвестна, недопустима или имеет недостающие обязательные аргументы, программа должна вывести сообщение об ошибке Invalid inputи предложить ввести новую команду.

Если операция не удалась, программа должна вывести сообщение Operation failedи запросить новую команду.

Команды
Команды навигации и рабочего каталога
up— Перейдите на один уровень выше в каталоге
up
Поведение:

Перемещается на один уровень выше по директории от текущей рабочей директории.
Если вы уже находитесь в корневом каталоге, ничего не происходит (ошибки не возникает).
После успешного перехода выводится новый путь к текущей рабочей директории.
cd— Перейти в указанный каталог
cd path_to_directory
path_to_directory— относительный или абсолютный путь для перехода ( обязательно )
Поведение:

Переходит в указанный каталог.
Может принимать как относительные, так и абсолютные пути.
Если путь не существует или не является каталогом, выводится сообщение Operation failed, и программа остается в текущем каталоге.
В случае успеха выводит новый путь к текущей рабочей директории.
ls— Отображение списка файлов и каталогов в текущем каталоге
ls
Выход:

Список всех файлов и папок в текущем каталоге.
Сначала перечислены папки, затем файлы, все в алфавитном порядке.
Каждая запись содержит имя (с расширением для файлов) и тип (файл или папка).
Пример:

folder1    [folder]
folder2    [folder]
file1.txt  [file]
file2.md   [file]
Команды обработки данных
1. csv-to-json— Преобразование CSV в JSON
Преобразуйте CSV-файл в JSON-файл с помощью Streams.

csv-to-json --input data.csv --output data.json
--input— путь к входному CSV-файлу ( обязательно )
--output— путь к выходному JSON-файлу ( обязательно )
Поведение:

Первая строка CSV-файла рассматривается как заголовок.
Каждая последующая строка становится JSON-объектом, в котором в качестве ключей используются имена заголовков.
Выходной файл должен содержать JSON-массив объектов.
Необходимо использовать конвейер Readable Stream → Transform Stream → Writable Stream.
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл не существует, вывести сообщение.Operation failed
Пример:

Вход data.csv:

name,age,city
Alice,30,New York
Bob,25,London
Выход data.json:

[
  { "name": "Alice", "age": "30", "city": "New York" },
  { "name": "Bob", "age": "25", "city": "London" }
]
2. json-to-csv— Преобразование JSON в CSV
Преобразуйте JSON-файл (массив объектов) в CSV-файл с помощью потоков.

json-to-csv --input data.json --output data.csv
--input— путь к входному JSON-файлу ( обязательно )
--output— путь к выходному CSV-файлу ( обязательно )
Поведение:

Входными данными должен быть JSON-массив объектов.
Первая строка вывода содержит заголовки (ключи из первого объекта).
Каждый объект становится строкой CSV-файла.
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл не существует или содержит недопустимый JSON, вывести сообщение.Operation failed
3. count— Подсчет строк, слов и символов в текстовом файле.
Подсчет строк, слов и символов в файле (аналогично команде wc).

count --input file.txt
--input— путь к входному файлу ( обязательно )
Формат вывода:

Lines: 42
Words: 350
Characters: 2048
Поведение:

Для обработки файла необходимо использовать API Streams (не загружайте весь файл в память).
Слово — это любая последовательность символов, не являющихся пробелами.
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл не существует, вывести сообщение.Operation failed
4. hash— Вычислить хеш файла
Вычислите криптографический хеш файла.

hash --input file.txt
hash --input file.txt --algorithm md5
hash --input file.txt --save
--input— путь к входному файлу ( обязательно )
--algorithm— Алгоритм хеширования для использования (необязательно, по умолчанию: sha256). Поддерживаемые значения: sha256, md5,sha512
--save— необязательный флаг; если указан, сохранить хеш в файл, расположенный рядом с исходным файлом.
Формат вывода:

sha256: 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824
Поведение:

Необходимо использовать crypto.createHashс Streams API.
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл не существует, вывести сообщение.Operation failed
Если алгоритм не поддерживается, вывести сообщение.Operation failed
Если --saveпередан параметр, запишите хеш в <inputFilename>.<algorithm>(пример: file.txt.sha256)
5. hash-compare— Сравните хеш файла с ожидаемым хешем.
Вычислите хеш файла и сравните его со значением, хранящимся в хеш-файле.

hash-compare --input file.txt --hash file.txt.sha256
hash-compare --input file.txt --hash file.txt.md5 --algorithm md5
--input— путь к входному файлу ( обязательно )
--hash— путь к файлу с ожидаемым хешем ( обязательно )
--algorithm— Алгоритм хеширования для использования (необязательно, по умолчанию: sha256). Поддерживаемые значения: sha256, md5,sha512
Формат вывода:

OK
или

MISMATCH
Поведение:

Необходимо вычислить хеш при --inputиспользовании API потоков.
Необходимо считать ожидаемое значение хеша из --hashфайла.
Сравнение должно быть нечувствительным к регистру и игнорировать завершающий символ новой строки в хеш-файле.
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл или хеш-файл не существует, вывести сообщение.Operation failed
Если алгоритм не поддерживается, вывести сообщение.Operation failed
6. encrypt— Зашифровать файл
Зашифровать файл с помощью AES-256-GCM.

encrypt --input file.txt --output file.txt.enc --password mySecret
--input— путь к входному файлу ( обязательно )
--output— путь к зашифрованному выходному файлу ( обязательно )
--password— Пароль, используемый для получения ключа шифрования ( обязательно )
Формат выходного файла (двоичный):

Первые 16 байт:salt
Следующие 12 байт:iv
Затем:ciphertext
Последние 16 байт:authTag
Поведение:

Необходимо получить 32-байтовый ключ из passwordиsalt
Необходимо выполнить шифрование с использованиемAES-256-GCM
Необходимо использовать API Streams от начала до конца.
Запрещается загружать весь файл в память. Допускается только буферизация в памяти:
заголовок (первые 28 байт = salt+ iv)
Аутентификационный тег (последние 16 байт)
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл не существует, вывести сообщение.Operation failed
7. decrypt— Расшифровка файла
Расшифровать файл, созданный программой encrypt.

decrypt --input file.txt.enc --output file.txt --password mySecret
--input— путь к зашифрованному входному файлу ( обязательно )
--output— путь к выходному файлу ( обязательно )
--password— Пароль, используемый для получения ключа шифрования ( обязательно )
Поведение:

Необходимо разобрать salt(первые 16 байт) и iv(следующие 12 байт) из входных данных.
Необходимо выполнить разбор authTag(последних 16 байт) из входных данных.
Необходимо выполнить расшифровку с использованием AES-256-GCMпроверки аутентификационного тега.
Необходимо использовать API Streams от начала до конца.
Расшифрованный результат должен точно соответствовать содержимому исходного файла.
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл не существует или аутентификация не удалась, вывести сообщение.Operation failed
8. log-stats— Анализ большого файла журнала с использованием рабочих потоков.
Вычисление статистических данных для большого файла журнала с использованием рабочих потоков для параллельной обработки.

log-stats --input logs.txt --output stats.json
--input— путь к входному файлу журнала ( обязательно )
--output— путь к выходному JSON-файлу ( обязательно )
Формат строки лога (разделенные пробелами):

<isoTimestamp> <level> <service> <statusCode> <responseTimeMs> <method> <path>
Пример строки:

2026-02-01T12:34:56.789Z INFO user-service 200 123 GET /api/users
Формат вывода (JSON):

{
  "total": 1000,
  "levels": { "INFO": 700, "WARN": 200, "ERROR": 100 },
  "status": { "2xx": 800, "3xx": 50, "4xx": 120, "5xx": 30 },
  "topPaths": [
    { "path": "/api/users", "count": 120 },
    { "path": "/api/orders", "count": 95 }
  ],
  "avgResponseTimeMs": 137.42
}
Поведение:

Разделите входной файл на N фрагментов (где N = количество ядер процессора), убедившись, что фрагменты начинаются и заканчиваются на границах строк.
Каждый фрагмент данных отправляется в рабочий поток для анализа и частичной агрегации.
Каждый обработчик возвращает частичную статистику: количество запросов по уровням, количество запросов по классам статуса, количество путей, общее количество строк, суммарное время ответа.
Основной поток объединяет частичные статистические данные и вычисляет итоговую статистику.avgResponseTimeMs
Запишите результат в формате JSON в выходной файл.
Для параллельной обработки необходимо использовать рабочие потоки (Worker Threads).
Количество рабочих процессов должно равняться количеству логических ядер ЦП.
Пути указываются относительно текущего рабочего каталога или могут быть абсолютными.
Если входной файл не существует, вывести сообщение.Operation failed
Генератор тестовых данных: Используйте предоставленный скрипт для создания большого файла журнала для тестирования:

node scripts/generate-logs.js --output workspace/logs.txt --lines 500000
Структура проекта
src/
  main.js          — entry point, sets up REPL, handles navigation state
  repl.js          — REPL handler, command parsing and dispatching
  navigation.js    — navigation commands (up, cd, ls)
  commands/
    csvToJson.js   — csv-to-json command handler
    jsonToCsv.js   — json-to-csv command handler
    count.js       — count command handler
    hash.js        — hash command handler
    hashCompare.js — hash-compare command handler
    encrypt.js     — encrypt command handler
    decrypt.js     — decrypt command handler
    logStats.js    — log-stats command handler
  workers/
    logWorker.js   — worker thread for log-stats command
  utils/
    pathResolver.js  — resolve paths relative to current working directory
    argParser.js     — parse command line arguments
Подсказки
Используйте readlineмодуль для интерактивного ввода.
Используйте stream.pipeline(из stream/promises) для соединения потоков и надлежащей обработки ошибок.
При обработке CSV-файлов в потоке преобразования первую строку (заголовки) следует обрабатывать отдельно от строк данных.
Для этого json-to-csvвам потребуется буферизовать входные данные JSON для их анализа, а выходные данные CSV записывать в виде потока.
Для этого log-statsубедитесь, что фрагменты начинаются/заканчиваются на границах строк, чтобы избежать частичного заполнения строк лога.
Для объединения статистики, суммирования счетчиков и объединения карт путей перед вычислениемtopPaths
Перед выполнением операций всегда указывайте пути к файлам относительно текущего рабочего каталога.
Используется path.resolve()для объединения текущего рабочего каталога с относительными путями.
Использование process.cwd()здесь НЕцелесообразно — поддерживайте собственную переменную текущего рабочего каталога.
Сохраняйте текущий рабочий каталог в качестве состояния приложения на протяжении всей сессии.
